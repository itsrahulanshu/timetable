üîß Step 1: Understanding Environment Variables

Environment variables are configuration values that your app needs to function. You'll need to set these up in Vercel.

Required Environment Variables

Variable Name Description Example Required?
LPU_USERNAME Your LPU registration number 12345678 REQUIRED
LPU_PASSWORD Your LPU UMS password YourPassword@123 REQUIRED
ANTICAPTCHA_API_KEY API key from anti-captcha.com a1b2c3d4e5f6... REQUIRED

Optional Environment Variables

Variable Name Description Default Value
ONESIGNAL_APP_ID OneSignal app ID for push notifications Pre-configured (you can use your own)
ONESIGNAL_API_KEY OneSignal API key for notifications Pre-configured (you can use your own)
APP_URL Your Vercel deployment URL Will be set after deployment
AUTO_REFRESH_ENABLED Enable automatic timetable refresh true
AUTO_REFRESH_INTERVAL Refresh interval in minutes 30
PWA_VERSION App version for cache busting 1.9.4
üí° Tip: You can use the pre-configured OneSignal credentials initially, or create your own free account at OneSignal for custom push notifications.

üîë Step 2: Getting Your API Keys

1Get Anti-Captcha API Key

  1. Visit anti-captcha.com
  2. Create a free account
  3. Add credits to your account (minimum $5 recommended, lasts for months)
  4. Go to Settings ‚Üí API Setup
  5. Copy your API key
‚ö†Ô∏è Important: Anti-Captcha is a paid service but very affordable. $5 credit typically lasts for several months of regular use.

2(Optional) Setup OneSignal

  1. Visit onesignal.com and create free account
  2. Create a new app and select Web Push
  3. Choose Typical Setup
  4. For site URL, use your Vercel URL (you'll get this after deployment)
  5. Copy your App ID and API Key from Settings
üí° Note: OneSignal is optional. The app works fine without it, but you won't receive push notifications for timetable changes.

üì¶ Step 3: Push Code to GitHub

1Create GitHub Repository

  1. Go to github.com/new
  2. Create a new repository (e.g., "lpu-timetable")
  3. Choose Public or Private
  4. Don't initialize with README (we already have files)
  5. Click Create repository

2Push Your Code

Open terminal in your project folder and run:

# Initialize git repository git init # Add all files git add . # Commit files git commit -m "Initial commit - LPU Timetable App" # Add your GitHub repository (replace with your URL) git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git # Push to GitHub git branch -M main git push -u origin main
‚ö†Ô∏è Security Note: Make sure your .env file is NOT pushed to GitHub. It's already in .gitignore to prevent this.

üöÄ Step 4: Deploy to Vercel

1Connect Vercel to GitHub

  1. Go to vercel.com
  2. Click Sign Up and choose Continue with GitHub
  3. Authorize Vercel to access your GitHub account
  4. Click Add New Project
  5. Select your lpu-timetable repository

2Configure Project Settings

Vercel will auto-detect the project settings. Just verify:

  • Framework Preset: Other (or None)
  • Build Command: (leave empty or use default)
  • Output Directory: (leave empty)
  • Install Command: npm install

3Add Environment Variables

Before deploying, add your environment variables:

  1. In Vercel project setup, find Environment Variables section
  2. Add each variable one by one:
# Required Variables LPU_USERNAME=your_registration_number LPU_PASSWORD=your_password ANTICAPTCHA_API_KEY=your_api_key # Optional Variables (recommended) AUTO_REFRESH_ENABLED=true AUTO_REFRESH_INTERVAL=30 PWA_VERSION=1.9.4 # If you have your own OneSignal account ONESIGNAL_APP_ID=your_onesignal_app_id ONESIGNAL_API_KEY=your_onesignal_api_key
üí° Tip: For each variable, select "All" under environments (Production, Preview, Development) to ensure it works everywhere.

4Deploy!

  1. Click Deploy button
  2. Wait 1-2 minutes while Vercel builds and deploys your app
  3. You'll get a URL like: https://your-app.vercel.app
  4. Click the URL to open your timetable app!
üéâ Congratulations! Your app is now live on the internet!

5Update APP_URL (Optional)

After deployment, update the APP_URL for better notifications:

  1. Go to your Vercel project ‚Üí Settings ‚Üí Environment Variables
  2. Add new variable: APP_URL
  3. Value: https://your-app.vercel.app (your actual URL)
  4. Click Save
  5. Go to Deployments tab and click Redeploy

üîç Troubleshooting

‚ùå Deployment Failed

Solution:

  • Check if all required environment variables are set
  • Make sure there are no typos in variable names
  • Check Vercel deployment logs for specific errors

‚ùå Timetable Not Loading

Solution:

  • Verify your LPU_USERNAME and LPU_PASSWORD are correct
  • Check if Anti-Captcha API key is valid and has balance
  • Check browser console for errors (F12 ‚Üí Console)
  • Try the manual refresh button

‚ùå Notifications Not Working

Solution:

  • Make sure you allowed notifications in your browser
  • Check if ONESIGNAL_APP_ID and ONESIGNAL_API_KEY are set
  • Update APP_URL to your actual Vercel URL
  • Test notifications using the API endpoint: /api/test-notification

‚ùå App Shows Old Data

Solution:

  • Click the refresh button (üîÑ) in the app
  • Clear browser cache and reload
  • Check if AUTO_REFRESH_ENABLED is set to true

‚ùì Frequently Asked Questions

Q: Can multiple users use the same deployment?

A: No, this deployment is configured for a single user (your LPU account). Each user needs their own deployment with their own credentials.

Q: Will my password be visible to others?

A: No! Environment variables in Vercel are encrypted and not visible in the code. They're stored securely on Vercel's servers.

Q: How often does the timetable refresh?

A: By default, every 30 minutes. You can change this by updating the AUTO_REFRESH_INTERVAL environment variable.

Q: What happens if Anti-Captcha runs out of credits?

A: The login will fail and timetable won't update. You'll need to add more credits to your Anti-Captcha account.

Q: Can I use a custom domain?

A: Yes! Vercel supports custom domains. Go to Project Settings ‚Üí Domains to add one.

Q: Is my data stored anywhere?

A: The app uses Vercel's serverless functions. Your timetable data is temporarily cached for better performance but not permanently stored anywhere except Vercel's ephemeral storage.
